# Spring Reference

## Spring Concepts and Annotations

| Concept / Annotation    | Usage in project                                                                                                                                                                     | Links to online resources                                                                                                                   |
| ----------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------- |
| @SpringBootApplication  | Placed on CoreApplication.kt to bootstrap the entire application. Enables auto-configuration, component scanning, and Spring Boot defaults.                                          | [https://docs.spring.io/spring-boot/reference/using/using-the-springbootapplication-annotation.html]docs.spring                             |
| @Configuration          | Used in OpenApiConfig.kt to mark the class as a source of bean definitions. Allows defining Spring-managed beans via @Bean methods.                                                  | [https://docs.spring.io/spring-framework/docs/current/javadoc-api/org/springframework/context/annotation/Configuration.html]docs.spring     |
| @Bean                   | Used inside OpenApiConfig.kt to register the OpenAPI configuration object as a Spring-managed bean.                                                                                  | [https://docs.spring.io/spring-framework/reference/core/beans/annotation-config/bean.html]spring                                            |
| @RestController         | Placed on EmployeeController and LearningMaterialController to mark them as REST endpoints. Combines @Controller and @ResponseBody so return values are serialized directly to JSON. | [https://docs.spring.io/spring-framework/docs/current/javadoc-api/org/springframework/web/bind/annotation/RestController.html]howtodoinjava |
| @RequestMapping         | Set at class level on both controllers with the base path /api. All handler methods inherit this prefix.                                                                             | [https://docs.spring.io/spring-framework/reference/web/webmvc/mvc-controller/ann-requestmapping.html]docs.spring                            |
| @GetMapping             | Used on read endpoints in both controllers, e.g. GET /api/employees and GET /api/employees/{id}. Maps HTTP GET requests to the annotated handler method.                             | [https://docs.spring.io/spring-framework/docs/current/javadoc-api/org/springframework/web/bind/annotation/GetMapping.html]docs.spring       |
| @PostMapping            | Used on create endpoints in both controllers, e.g. POST /api/employees. Maps HTTP POST requests to the annotated handler method.                                                     | [https://docs.spring.io/spring-framework/docs/current/javadoc-api/org/springframework/web/bind/annotation/PostMapping.html]docs.spring      |
| @PutMapping             | Used on update endpoints in both controllers, e.g. PUT /api/employees/{id}. Maps HTTP PUT requests to the annotated handler method.                                                  | [https://docs.spring.io/spring-framework/docs/current/javadoc-api/org/springframework/web/bind/annotation/PutMapping.html]leyaa             |
| @DeleteMapping          | Used on delete endpoints in both controllers, e.g. DELETE /api/employees/{id}. Maps HTTP DELETE requests to the annotated handler method.                                            | [https://docs.spring.io/spring-framework/docs/current/javadoc-api/org/springframework/web/bind/annotation/DeleteMapping.html]leyaa          |
| @PathVariable           | Extracts the id segment from the URL in get-by-id, update, and delete endpoints in both controllers. Binds the URI template variable to the method parameter.                        | [https://docs.spring.io/spring-framework/docs/current/javadoc-api/org/springframework/web/bind/annotation/PathVariable.html]howtodoinjava   |
| @RequestBody            | Used on the employee and learningMaterial parameters in create and update endpoints. Deserializes the incoming JSON request body into the Kotlin data class.                         | [https://docs.spring.io/spring-framework/docs/current/javadoc-api/org/springframework/web/bind/annotation/RequestBody.html]howtodoinjava    |
| @Tag (OpenAPI)          | Applied at controller class level to group related endpoints under a named tag in Swagger UI (e.g. "Employee", "Learning Material").                                                 | [https://springdoc.org/#springdoc-openapi-3-0-0-early-access]springdoc                                                                      |
| @Operation (OpenAPI)    | Applied to each handler method to provide a human-readable summary and description visible in Swagger UI. All five CRUD endpoints in each controller are annotated.                  | [https://springdoc.org/#springdoc-openapi-3-0-0-early-access]springdoc                                                                      |
| @ApiResponses (OpenAPI) | Wraps a list of @ApiResponse annotations on each handler method to document all possible HTTP response codes.                                                                        | [https://springdoc.org/#springdoc-openapi-3-0-0-early-access]springdoc                                                                      |
| @ApiResponse (OpenAPI)  | Documents individual response codes (e.g. 200, 201, 404) per endpoint, including the content type and schema of the response body.                                                   | [https://springdoc.org/#springdoc-openapi-3-0-0-early-access]springdoc                                                                      |
| @Content (OpenAPI)      | Nested inside @ApiResponse to describe the media type (application/json) and link to the response schema for success responses.                                                      | [https://springdoc.org/#springdoc-openapi-3-0-0-early-access]springdoc                                                                      |
| @Schema (OpenAPI)       | Used both at class level on Employee, LearningMaterial, and LearningMaterialType models and on individual fields to provide descriptions and examples in the generated API docs.     | [https://springdoc.org/#springdoc-openapi-3-0-0-early-access]springdoc                                                                      |


## Libraries and Dependencies


| Library / Dependency                      | Usage in project                                                                                                                                                                                          | Links to online resources                                                                                                                                                     |
| ----------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| spring-boot-starter-parent 4.0.3          | Declared as the Maven parent POM. Provides dependency management, default plugin versions, and build conventions for the whole project.                                                                   | [https://docs.spring.io/spring-boot/docs/current/reference/html/using.html#using.build-systems.starters.parent-pom]springboottutorial                                         |
| spring-boot-starter-webmvc                | Core web dependency that brings in Spring MVC, an embedded servlet container, and JSON support. Powers all REST controllers in the project.                                                               | [https://docs.spring.io/spring-boot/docs/current/reference/html/web.html#web.servlet.spring-mvc]libraries                                                                     |
| kotlin-reflect                            | Required by Spring to introspect Kotlin classes at runtime. Enables features like constructor injection and data class binding to work correctly.                                                         | [https://kotlinlang.org/api/latest/jvm/stdlib/kotlin.reflect/]spring                                                                                                          |
| kotlin-stdlib                             | The Kotlin standard library. Provides core language extensions, collections, and utilities used throughout the Kotlin source code.                                                                        | [https://kotlinlang.org/api/latest/jvm/stdlib/]spring                                                                                                                         |
| jackson-module-kotlin                     | Enables Jackson to serialize and deserialize Kotlin data classes, including support for default parameter values and non-nullable types. Used when converting Employee and LearningMaterial to/from JSON. | [https://github.com/FasterXML/jackson-module-kotlin]mvnrepository                                                                                                             |
| springdoc-openapi-starter-webmvc-ui 3.0.1 | Generates the OpenAPI 3 spec and serves Swagger UI at /swagger-ui.html. Picks up all @Operation, @Schema, and related annotations automatically.                                                          | [https://springdoc.org/]springdoc                                                                                                                                             |
| spring-boot-starter-webmvc-test           | Provides MockMvc and Spring test context support for writing integration-style controller tests.                                                                                                          | [https://docs.spring.io/spring-boot/docs/current/reference/html/features.html#features.testing.spring-boot-applications.testing-with-running-server.web-environment]libraries |
| kotest-runner-junit5 5.9.1                | Enables Kotest specs to run on the JUnit 5 platform used by Maven Surefire. All project tests are written in Kotest FunSpec style.                                                                        | [https://kotest.io/docs/framework/runner/junit5.html]howtodoinjava                                                                                                            |
| kotest-extensions-spring 1.3.0            | Integrates the Spring test context (e.g. @SpringBootTest) with Kotest test lifecycle. Allows Spring beans to be injected into Kotest specs.                                                               | [https://github.com/kotest/kotest-extensions-spring]howtodoinjava                                                                                                             |
| mockk-jvm 1.13.13                         | Kotlin-native mocking library used to create mocks and stubs in unit tests. Works well with Kotlin's non-nullable type system and coroutines.                                                             | [https://mockk.io/]howtodoinjava                                                                                                                                              |
| springmockk 4.0.2                         | Bridges MockK with Spring's test context, providing @MockkBean as the Kotlin equivalent of @MockBean. Used to replace Spring beans with mocks in controller tests.                                        | [https://springmockk.io/]howtodoinjava                                                                                                                                        |
| spring-boot-maven-plugin                  | Packages the application as an executable JAR. Also used to run the app locally via ./mvnw spring-boot:run.                                                                                               | [https://docs.spring.io/spring-boot/docs/current/maven-plugin/reference/htmlsingle/]springboottutorial                                                                        |
| spotless-maven-plugin 3.2.1               | Enforces consistent code formatting using KtLint 1.8.0. Run via ./mvnw spotless:apply to auto-format all Kotlin source files.                                                                             | [https://github.com/diffplug/spotless/tree/main/plugin-maven#kotlin]howtodoinjava                                                                                             |
| kotlin-maven-plugin 2.2.21                | Compiles Kotlin sources as part of the Maven build lifecycle. Configured with the allopen compiler plugin to support Spring's proxy-based features.                                                       | [https://kotlinlang.org/docs/maven.html]spring                                                                                                                                |
| kotlin-maven-allopen 2.2.21               | Makes annotated Kotlin classes open at compile time without requiring the open keyword. Necessary because Spring needs to subclass beans for proxying.                                                    | [https://github.com/JetBrains/kotlin/tree/master/plugins/allopen-maven-plugin]spring                                                                                          |
